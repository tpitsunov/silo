# Безопасность и подтверждения

Безопасность — главный приоритет SILO. Если вы поместите API-ключ в обычный промпт или контекст, модель может случайно отправить его в сеть, выдать в ответе или записать в файл.

SILO хранит секреты в защищенном "сейфе", к которому у LLM нет прямого доступа.

## Класс `Secret`

Используйте `Secret.require()` для безопасного получения учетных данных внутри Python-кода, не заставляя LLM передавать их.

```python
from silo import Skill, Secret, JSONResponse

app = Skill("github_skill")

@app.command()
def get_profile():
    token = Secret.require("GITHUB_TOKEN")
    # Процедурная логика здесь...
    return JSONResponse({"status": "success"})
```

### Как `Secret.require` ищет ключи:
Когда команде нужен токен:

1. **Переменные окружения**: SILO сначала проверяет `os.environ`.
2. **Системный Keychain**: SILO запрашивает хранилище ОС (Keychain на macOS, Credential Locker на Windows, Secret Service на Linux). Это самое надежное место.
3. **Интерактивный ввод**: Если ключа нет:
    - SILO открывает вкладку в браузере с формой ввода (если есть графический интерфейс).
    - Вы вводите ключ, он сохраняется в Keychain и выполнение продолжается. *Ключ никогда не печатается в логах.*
4. **Отказ в Headless-режиме**: Если SILO запущен в фоне (например, ИИ-агентом) и ключа нет, он вернет структурированную ошибку `{"error": "SILO_AUTH_REQUIRED"}`.

## Подтверждения (Human-in-the-Loop)

Некоторые действия (удаление базы, коммит кода, отправка письма) не должны выполняться ИИ полностью автономно.

SILO позволяет требовать подтверждение для конкретных команд.

```python
from silo import Skill

app = Skill("db_manager")

@app.command(require_approval=True)
def drop_table(table_name: str):
    """Удаляет таблицу в базе данных."""
    return {"status": "table dropped"}
```

Когда агент запускает такую команду, выполнение приостанавливается:
1. SILO ищет терминал. Если он есть, запрашивает подтверждение `[y/N]`.
2. Если терминала нет, но есть браузер, открывается окно подтверждения с деталями команды.
3. Если пользователь отказал или интерфейс недоступен, агент получит ошибку: `{"error": "SILO_APPROVAL_REQUIRED"}`.
